Komentáře v Pythonu

Komentáře jsou části kódu, které Python při spuštění ignoruje. Slouží programátorům k několika účelům:

  • Vysvětlení kódu: Pomáhají objasnit, co dělá složitější nebo méně zřejmá část programu.
  • Poznámky pro sebe nebo ostatní: Můžete si zanechat připomínky, nápady na vylepšení nebo varování.
  • Dočasné "vypnutí" kódu: Pokud chcete část kódu dočasně deaktivovat (např. při ladění), můžete ji zakomentovat místo mazání.

Jednořádkové komentáře

V Pythonu jednořádkový komentář začíná znakem mřížky (#). Vše od znaku # až do konce řádku je považováno za komentář.


# Toto je jednořádkový komentář. Python ho úplně přeskočí.
x = 10  # Toto je také komentář, může být na stejném řádku jako kód.
# y = 20  Tento řádek kódu se neprovede, protože je zakomentovaný.
print(x) # Vypíše hodnotu proměnné x

# Příklad z examples/CV2_ovoce_a_zelenina_Barcot_1-33.py
# #Domácí úkol 2 - ovoce a zelenina <- Toto je komentář na začátku souboru

Víceřádkové komentáře (Docstringy)

Python nemá speciální syntaxi pro víceřádkové komentáře ve stejném smyslu jako jednořádkové s #. Často se však pro víceřádkové komentáře nebo dokumentační řetězce (tzv. "docstringy") používají řetězce uzavřené do trojitých uvozovek ("""...""" nebo '''...''').

Pokud takový řetězec není přiřazen žádné proměnné a je na začátku modulu, funkce nebo třídy, stává se z něj docstring, který slouží k dokumentaci.


"""
Toto je víceřádkový řetězec,
který může sloužit jako komentář
nebo jako docstring.
Python ho technicky zpracuje jako řetězec,
ale pokud není nikam přiřazen, tak se "ztratí".
"""
def moje_funkce():
    """Toto je docstring pro moji_funkci.
    Vysvětluje, co funkce dělá.
    """
    print("Funkce byla zavolána.")

# print("Tento text se vypíše")
'''
Tento blok kódu je také "zakomentován"
pomocí víceřádkového řetězce.
print("Toto se nevypíše.")
print("Ani toto.")
'''
print("Pokračujeme dál.")

Pro běžné komentování kódu během vývoje je však preferováno používání # na začátku každého řádku komentáře.


# Toto je preferovaný způsob
# pro víceřádkový komentář,
# pokud to není docstring.

Dobré komentáře dělají kód srozumitelnějším a snadněji udržovatelným, ale snažte se psát kód tak, aby byl co nejvíce samovysvětlující. Komentujte spíše "proč" něco děláte, než "co" děláte, pokud je to z kódu zřejmé.
