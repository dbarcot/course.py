Víceřádkové komentáře a Docstringy

V Pythonu se pro víceřádkové komentáře nebo pro dokumentaci kódu (tzv. "docstringy") používají řetězce uzavřené do trojitých uvozovek (buď trojité dvojité uvozovky """...""" nebo trojité jednoduché uvozovky '''...''').

Víceřádkové komentáře (ne jako Docstring)

Pokud trojitě uvozený řetězec není prvním příkazem v modulu, funkci, třídě nebo metodě, Python ho považuje za běžný víceřádkový řetězec. Pokud tento řetězec není nikam přiřazen, Python ho při provádění kódu v podstatě ignoruje, a proto se často používá jako víceřádkový komentář.


"""
Toto je víceřádkový komentář.
Může se rozprostírat přes
několik řádků.
"""

'''
I toto je víceřádkový komentář,
používající jednoduché uvozovky.
'''

print("Tento řádek se provede.")

"""Tento řetězec na stejném řádku
se také chová jako komentář, pokud není přiřazen."""

Docstringy (Dokumentační řetězce)

Pokud je trojitě uvozený řetězec prvním příkazem v modulu, funkci, třídě nebo metodě, Python ho speciálně zpracuje jako "docstring" (dokumentační řetězec). Docstringy jsou důležitou součástí dokumentace kódu a jsou přístupné za běhu programu (např. pomocí funkce help() nebo v interaktivním prostředí).


def moje_funkce(parametr1, parametr2):
    """Tento docstring popisuje funkci.

    Vysvětluje, co funkce dělá, jaké má parametry
    a co vrací.

    Args:
        parametr1 (int): Popis prvního parametru.
        parametr2 (str): Popis druhého parametru.

    Returns:
        bool: Popis návratové hodnoty.
    """
    # Zde je tělo funkce
    print(f"Parametr 1: {parametr1}, Parametr 2: {parametr2}")
    return True

# Příklad použití docstringu
# help(moje_funkce) # Vypíše docstring funkce

class MojeTrida:
    """Docstring pro třídu MojeTrida."""

    def __init__(self, hodnota):
        """Docstring pro metodu __init__ (konstruktor)."""
        self.hodnota = hodnota

    def moje_metoda(self):
        """Docstring pro metodu moje_metoda."""
        print(f"Hodnota: {self.hodnota}")

Používání docstringů je silně doporučeno pro dokumentaci vašeho kódu, protože pomáhá vám i ostatním rychle pochopit účel a použití různých částí programu.
